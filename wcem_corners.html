Working Corner/Edge Method
Testing a Corner Solving Strategy

There are many ways to solve the corners in the top layer using the Working Corner/Edge Method. To test a strategy on all possible configurations (programmatically) it can be defined as a list of sequences and instructions on how they are used. The following is an example definition of a strategy and the results of using it on all possible configurations.

Sequences

The following sequences are used to solve the working corner (cubie in position Front-Right-Down = FRD). Only the centers, working corner and matching edge are shown for clarity.

FRD -> LUF
FRD -> UFL
FRD -> UBR
FRD -> RUB
FRD -> URF
FRD -> FUR
FRD -> BUL
FRD -> ULB

For the above cubes with 5-move sequences the last move would not normally be done, instead it would be to select a cubie to be the next working corner by twisting the top layer to move it to the location of the previous one (one of the two grays on cube below). That is also the next step for cubes with 3-move sequences. The last step is to move the cubie (in gray location) to the working corner by using the appropriate inverse 3-move sequence.



The following are used to solve configurations where two cubies are misoriented and will be referred to as "Solve Sequences". In most cases the whole cube will have to be turned to position the two cubies in the locations shown (orientations must also match), then it is solved normally using the above sequences and the instructions below, with one minor exception which is that the working corner is moved to the location of a solved cubie (moves 1-3 on cubes below) as opposed to an unsolved one.


Instructions

1. If only two cubies are misoriented then use one of the Solve Sequences.

2. If the working corner is from the top layer and is oriented correctly (the bottom facelet is not the color of the top face) then solve it. Then twist the top layer to select another unsolved cubie to be the next working corner, move it there with inverse 3-move sequence. When selecting the cubie, pick one that will be oriented the right way if possible, otherwise it will have to be traded as described by the following.

3. If the working corner is oriented wrong or not from the top layer then trade it for another. To do that, twist the top layer to move an unsolved cubie, preferably the one that belongs in the bottom layer, to one of the two gray locations and then move the working corner there using one of the above 3-move sequences. Twist the top layer to select another unsolved cubie to be the next working corner, move it there with inverse 3-move sequence. When selecting the cubie, pick one that will be oriented the right way if possible.
