Working Corner/Edge Method
Testing an Edge Solving Strategy

There are many ways to solve the edges in the top layer using the Working Corner/Edge Method. To test a strategy on all possible configurations (programmatically) it can be defined as a list of sequences and instructions on how they are used. The following is an example definition of a strategy and the results of using it on all possible configurations.

Sequences

The following sequences are used to solve the working edge cubie (FR). Only the movement of the working edge cubie is relevant here, the others can be ignored (they don't normally all get solved until the end).


FR -> UF
FR -> FU
FR -> RU
FR -> UR
FR -> UL
FR -> LU
FR -> UB
FR -> BU


The following sequences are used to move a cubie from the top layer to the working edge. Only the movement of the indicated cubie is relevant in each case, the others can be ignored.


UF -> FR
UR -> RF
UL -> FR
UB -> FR


The following are used to solve configurations where two cubies are swapped and will be referred to as "Solve Sequences". An interesting feature of these sequences is that the inverses also work.


Instructions

Overview
Solve the UL and UB positions first, then UF and UR. Half of the configurations will require one of the above two Solve Sequences to complete the solution.

Details
1. For the first move, if both UL and UB positions are unsolved then if either (or both) can be solved by twisting the top face then do so, otherwise continue to next step.

2. If the working edge is the UL or UB cubie then solve it.

3. If the UL or UB position is unsolved then move a cubie that belongs in the UL or UB position to the working edge.

4. After the UL and UB positions are solved, if the working edge is the UF or UR cubie then solve it otherwise move a cubie that should be in the UF or UR position to the working edge.

5. Use the above two Solve Sequences where applicable.
